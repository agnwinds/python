Documentation Overview

Note:  Documentation is an ongoing effort, and is not up to date.

At present the main resources are as follows:

  • A draft version of a cookbook which describes how to install  and update Python and run a first model
  • A Sphinx repository, which ultimately will be the main documention for Python
    • This is part of the Python distribution.  All of the files are tracked in git
    • This is also available on Readthedocs here. Note this link may not be the correct one in the end)
    • This documentation needs to be updated when parameters have been changed, or if the yaml files have been updated.
    • Instructions on how to do the updates are locally is here
    • It requires a software package known as sphinx
    • The command to generate the repository is currently:
      • cd docs/sphinx/; make html
  • A pydocs repository which provides information about the python scripts contained in $PYTHON/py_progs
    • These files are also tracked by git. 
    • The documentation should be updated if new python routines are added to py_progs, or if modifications to the _doc_ strings are made in existing programs
    • To regenerate them, use the routine write_docs.py which is in py_progs. 
    • Issue the following command from the docs/pydocs directory. 
      • ../../py_progs/write_docs.py ../../py_progs
    • No special sofware is required, except that the package MarkupPy may need to be installed
      • pip install MarkupPy
  • A Doxygen repository, which provides information about the source code.
    • Unlike the other documentation, the Doxygen information is not part of the repository.   Therefore if you want this, one will have to run software to generate the documeantions
    • To do this, one must go to the docs/doxygen directory and run
      • doxygen python_docs.config
    • Of course the assumes that you have doxygen
      • The Doxygen home page is here. 
      • To get the graphs that show which routines call which other routines, you also need to have the dot tool from Graphviz installed as well. 


Comments:

  • The S/W discussed above is readily available on various package distributions, e.g Homebrew or Fink or MacPorts for Macs or one can use conda.  Presumably, various linux package managers contain as well.  
    • The Homebrew command for Sphinx is simply "brew install Sphinx"
    • The Homebrew command for doxygen is simply "brew install doxygen"
    • The Homebrew command for GraphViz is similarly "brew install GraphViz"
  • Alternatively, one can go to  the home pages and download and install using the instruction provided.
